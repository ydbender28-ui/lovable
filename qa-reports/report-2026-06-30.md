## Cross-Site Issue Summary

These bugs appeared across multiple or all test sites:

### 1. 🔴 Anchor scroll broken (ALL sites)
- **Affected:** All 3 sites, all nav links
- **Symptom:** Clicking nav links never scrolls to sections. The "Jump to section" toolbar dropdown also fails on Test 3.
- **Root cause hypothesis:** The generated React components use `<a href="#section-id">` but the SPA doesn't attach matching `id` attributes to section elements, or the scroll handler intercepts but loses the target reference.
- **User impact:** First impression navigation is completely broken for end users. This is the #1 issue.

### 2. 🔴 BeforeAfter.tsx null ref crash (Test 1)
- **Affected:** At minimum Test 1, likely any site with a BeforeAfter component
- **Symptom:** `Cannot read properties of null (reading 'getBoundingClientRect')` crashes the entire React tree, blanking the preview
- **Root cause:** `calcPct` calls `el.getBoundingClientRect()` without a null check on `el`
- **Fix:** Add `if (!el) return;` guard before the `getBoundingClientRect()` call in BeforeAfter.tsx

### 3. 🔴 Auto-fix parse failure
- **Affected:** At least Test 1
- **Symptom:** "Try to Fix" button triggered, AI returned a response starting with `<<<SEARCH>>>`, system returned "Error: Could not parse response"
- **Impact:** Critical failures cannot be self-healed; users are left with broken previews

### 4. 🟠 Systemic SyntaxError: `colors` re-declaration (181 projects)
- **Affected:** 181+ older project preview URLs
- **Symptom:** `Uncaught SyntaxError: Identifier 'colors' has already been declared`
- **Root cause:** A variable named `colors` is declared twice in the bundled output — likely a Tailwind or theming variable collision introduced by a recent code generation change

### 5. 🟠 Systemic SyntaxError: Invalid regex flags (146 projects)
- **Affected:** 146+ project preview URLs (different set from `colors` issue)
- **Symptom:** `Uncaught SyntaxError: Invalid regular expression flags`
- **Root cause:** Code generation emitting regex with unsupported flags (likely `v` flag not supported in older V8/Safari)

## Top 3 Priority Fixes

### P1 — Fix anchor scroll navigation (affects ALL generated sites)
Every site has broken nav. Root cause is in how React components render sections without matching IDs for the nav `href` anchors, or how the click handler is wired. This makes the product feel unfinished to any user clicking a nav link.

**Suggested fix:** Audit the Nav component template — ensure every nav link that uses `href="#features"` has a corresponding section rendered with `id="features"`. Alternatively, switch to `scrollIntoView` with `document.getElementById(target)` with a null guard.

### P2 — Fix BeforeAfter.tsx null ref crash
A component crash that blanks the entire preview is a critical user-facing failure. The fix is a two-line change — add a null guard to `calcPct`.

**Suggested fix:**
```tsx
// Before
const calcPct = (el, e) => {
  const rect = el.getBoundingClientRect();
// After  
const calcPct = (el, e) => {
  if (!el) return 0;
  const rect = el.getBoundingClientRect();
```

### P3 — Fix auto-fix parse failure
When a crash occurs, "Try to Fix" is the user's recovery path. If it silently fails with "Error: Could not parse response," the user is stuck with no recourse. The AI response format (`<<<SEARCH>>>`) suggests the fix response parser has a strict format requirement that the LLM occasionally misses.

**Suggested fix:** Make the fix parser more robust with fallback parsing, or surface a clearer error with a "retry" option rather than silent failure.
