## TOP PRIORITY FIXES NEEDED:

1. **Fix PricingTable.tsx template — SyntaxError at line 47:12 (blocks ALL sites with pricing)**: The AI-generated PricingTable component consistently crashes with `Unexpected token` at the same location in every build. Root cause: a `.map()` callback rendering a `<a>` element with a massively long inline `style={{...}}` attribute string that breaks the JSX parser. Fix: either split the style object into a variable, or fix the AI prompt to generate cleaner JSX. This breaks 2/3 test builds (Velvet Stone and TaskFlow) and likely breaks ALL sites with pricing sections.

2. **Fix Booking.tsx template — SyntaxError at line 89:14 (blocks ALL sites with reservation forms)**: Same pattern as #1 but in booking form components. An `<input>` element rendered inside a `.map()` produces an unexpected token crash. Breaks any site with a booking/reservation form (Terra Mia and any other spa/restaurant prompt).

3. **Fix "Try to Fix" — it always returns "Response was too long to complete"**: The auto-fix feature is completely non-functional. When a syntax error occurs, the fix request sends too much context (the full 19,000-33,000 char codebase) back to the AI, which then hits its response size limit. Fix: send only the broken file to the AI for repair, not the entire project codebase. Currently this means ALL syntax errors are permanently unrecoverable.

4. **Fix Enter key behavior in dashboard build textarea**: Pressing Enter should reliably create a new project AND navigate to it. Currently it sometimes creates the project but does not navigate, leaving the user stranded on the dashboard.

5. **Prevent ghost project creation**: When a build input interaction fails or is interrupted, do not create a nameless/empty project in the dashboard. Add a check that a project only persists if the build was actually submitted with valid content.
